[DSA] 資料結構介紹

陣列 Array

特性

  • 儲存有序串列之相同型態的資料於連續記憶空間 ➪ 新增、刪除慢
  • 須事先宣告固定的記憶體空間 ➪ 存取快 O(1)

串列 List

元素以線性方式（有序）儲存，且可重復。

類型

  • [**Linked list**](##鏈結串列 Linked list)（無序）
  • ArrayList：自動擴張容量

鏈結串列 Linked list

特性

  • 不用事先宣告連續的記憶體空間，資料數量為動態 ➪ 新增、刪除快
  • 存取 O(n) 節點 node 須從頭開始找 ➪ 存取慢

類型

  • 單向鏈結串列 Singly Linked List

    追加資料只須更動指標的指向

  • 雙向鏈結串列 Doubly Linked List

    追加或刪除資料時，須更動比單向鏈結串列更多的指標次數。

  • 環狀鏈結串列 Circular Linked List

    每個節點都可以是串列頭

用途

  • 無法預知、需頻繁變動資料的數量
  • 大量新增、刪除資料

集合 Set

特性

  • 無序、元素不重複

類型

  • HashSet（高效） ➪ 快速查找
  • LinkedHashSet（有序）➪ 快速查找
  • TreeSet（有序）➪ 確保元素處於有序

用途

  • 一堆可能重複的資料
  • 找出共通之元件

Map

鍵映射至值的集合

特性

  • 每一映射的鍵不重複
  • 每一鍵最多只能映射一值

類型

  • [**HashTable**](##雜湊表 Hash Table)：以雜湊函式 Hash Function 算出鍵值並放入對應的陣列索引，且不允許鍵值為 null。

  • HashMap（高效）：允許鍵值為 null

    • 新增 put
    • 刪除 remove()
    • 取得 get()
    • 查找指定鍵值 containsKey()、containsValue()

  • LinkedHashMap（有序）

  • TreeMap（有序）

堆疊 Stack

特性

  • 先進後出 FILO、後進先出 LIFO
  • 只能從頂端存取資料
  • 只需一個指標：top

實作

  • Array

  • Linked list

常見操作

  • push() 新增元素 O(1)
  • pop() 刪除頂端元素 O(1)
  • peek() 查看頂端元素

用途

  • 上一頁
  • 還原

佇列 Queue

特性

  • 先進先出 FIFO的有序表

  • 不易於插入或刪除某指定資料

  • 若要取出最新資料，需先移除前面所有資料。

  • 前端 front、尾端 rear（back）

特殊類型

  • Circular Queue：開頭結尾接起來 ➪ 解決 front 前面有未使用空間

    當 rear 到達 MAXSIZE 時，將其設為 0。

  • Deque（Double-ended queue、Head-tail linked list）：兩端都可放資料、拿資料

    • Input restricted deque：只有一邊可放入 ↔ Output restricted deque

  • Priority Queue：看誰有優先權 ➪ 不符合 FIFO　ex. 工作排程

  • Multiple Queue：兩個 queue 共用一 array

樹 Tree

名詞介紹

  • 節點 Node：每一資料元素
  • 枝 Branch：節點向下延伸擴展的枝
  • 根節點 Root：樹狀結構的最上層，具有唯一性。
  • 葉節點 Leaf：沒有子節點的節點 ↔ 非終端點 Non-terminal Nodes
  • 祖先節點 Ancenstors：某節點以上的節點 ↔ 子孫節點 Descendant
  • 父節點 Parent、子節點 Child：父節點向下分支出子節點
  • 兄弟節點 Siblings：同一父節點的其他節點
  • 分支度 Dregree：每一節點的節點數量，且每條分支可看作 Linked list。
  • 階層 Level：節點的層級數量（根的階層＝ 1）
  • 高度Height：某節點下最長邊之距離
  • 深度 Depth：根節點至某節點之距離（根的深度＝ 0）

特性

  • 非線性結構
  • 不能形成環路 ➪ 爸爸只有一位

二元樹 Binary tree

每一節點的子節點 ≤ 2，且分為左右子樹。

類型

  • 歪斜樹 Skewed binary tree：分支只有左或只有右

  • 嚴格二元樹 Strictly binary tree：每一節點的子節點只能為 0 或 2

  • 滿二元樹 Full binary tree（= Perfect binary tree）

    若階層為 k，節點數為 $2^k-1$。

  • 完全二元樹 Complete binary tree：最後一層以外皆全滿

    若階層為 k，$2^{(k-1)} \leq$ 節點數 $\leq 2^k-1$

  • 二元搜尋樹 Binary Search tree（= 有序二元樹）

    • 左子樹鍵值 < 樹根鍵值 < 右子樹鍵值

    • 左右子樹亦是二元搜尋樹

    • 每一鍵值皆不相同

    • AVL（Adelson-Velsky and Landis）樹

      每一節點的左右邊子節點高度差＝ -1～1 ➪ 搜尋更快

    • 紅黑樹 Red–Black tree ➪ 保持樹狀結構的平衡

      • 每一節點是🔴或⚫，且根為⚫。
      • 葉節點一定是⚫的 NULL
      • 🔴的子節點必為⚫ ➪ 不能🔴—🔴，但⚫的子節點無限制。
      • 從任何節點出發，其下至葉節點所有路徑的⚫數目要相同。

遍歷 Traversal

  • Pre-order 前序：根 → 左 → 右 ➪ 深度優先搜尋 Depth-First Search（DFS）
  • In-order 中序：左 → 根 → 右
  • Post-order 後序：左 → 右→ 根
  • Level-order 層序：一層層從左至右 ➪ 廣度優先搜尋 Breadth-First Search（BFS）

堆積 Heap

一種完全二元樹

類型

  • 最小堆積 min-Heap
    • 所有父節點的值 < 子節點
    • 根＝min
  • 最大堆積 Max-Heap
    • 所有父節點的值 > 子節點
    • 根＝MAX

雜湊表 Hash Table

名詞介紹

  • 雜湊函式 Hash Function：將資料打亂並混合，建立出該資料對應的雜湊值。
  • 桶 bucket：儲存資料的位置，每一位置對應唯一位址 bucket address。
  • 槽 slot：每一桶中的資料欄位
  • 碰撞 collision：對應到相同位址 ➪ 以連結串列的方式儲存
  • 溢位 overflow：資料經過雜湊函數運算後，雜湊值所指向的桶位址已滿，無法再儲存。

原則

  • 降低碰撞與溢位的產生
  • 不宜過於複雜，越容易計算越佳
  • 文字 → 數字的鍵值
  • 雜湊值均勻分布在每一桶 ➪ 均衡性

雜湊函數

  • 直接定址法

    以鍵值 k 或 k＋常數 c 作為雜湊值

  • 數字分析法

    提取鍵值中取值比較均勻的數字作為雜湊值

  • 除留餘數法

    將鍵字 ÷ 某個＜哈希表長度 m，所得餘數為雜湊值。

  • 分段疊加法

    依雜湊表地址位數，將關鍵字分成位數相等的幾部分並相加，雜湊值為捨棄最高位後的總和。

  • 平分取中法

    當鍵值部分分佈不均勻時，可求出其平方值，然後依需求取中間幾位作為雜湊值。

  • 偽隨機數法

    採用一偽隨機數當作雜湊函數

處理碰撞、溢位

  • 線性探測法 Linear Probing

    以線性方式往後找到空 bucket 為止，若後面 bucket 滿了，可循環到前面找。

  • 平方探測法 Quadratic Probing

    $(H(x)±i^2)\ mod \ b\ \ _{1 ≤ i ≤ \frac{(b-1)}{2}}$ ➪ 尋找其他空 bucket

  • 鏈結法 Chaining

    於每一 Bucket 中用 Linked List

  • 再雜湊法 Rehashing

    用其他雜湊函式計算另一雜湊函數地址，重覆計算到不再衝突。

用途

  • 關鍵字搜尋
  • 加／解密

圖 Graph

由數個頂點 vertex（節點 node）、邊 edge 所構成，且之間互相連結或關聯 ➪ $G=(V,E)$

名詞

  • 路徑 path：兩頂點經過的邊

  • 路徑長度 path length：該路徑上所有邊的數量

  • 簡單路徑 simple path：起點～終點的頂點不可重複（起點可＝終點）

  • 相鄰 adjacent : 兩頂點有一條邊相連

  • 分支度 degree

    • 有向圖中：入分支度 In-Degree ↔ 出分支度 Out-Degree
    • 無向圖中：一個頂點含有幾條邊

  • 完整圖 complete graph

    • 有向圖中：n 個頂點，會有 $n(n-1)$ 條邊
    • 無向圖中：n 個頂點，會有 $\frac{n(n-1)}{2}$ 條邊

  • 子圖 subgraph

類型

  • 有向圖 Directed graph：邊具有方向性 ➪ 單向

  • 無向圖 Undirected graph ➪ 雙向

  • 權重圖 Weighted graph：數值說明連接程度

表示法

  • 相鄰矩陣 Adjacency Matrix：兩頂點間有邊值為 1，無為 0。

  • 相鄰串列 Adjacency List：每一頂點以 Singly Linked List 串接相鄰頂點，無邊不處理。

[DSA] 時間、空間複雜度

2022-06-25

演算法 Algorithm

解決特定問題或提供某功用之有限指令／敘述之集合

  1. Input：≥ 0 個 input data

  2. Output：至少 1 個 output 結果

  3. Definiteness：每個指令 / 敘述皆 clear, unambiguous

  4. Finitense：結束於有限個步驟

    ▴Program 不一定有限

  5. Effectiveness：步驟合理、可執行

時間複雜度 Time Complexity

執行演算法所需時間，以 Big-O 呈現。

漸進符號 Asymptotic Notation

  • Big-O（Ο）

    演算法時間函式的上限 Upper bound ➪ 最壞狀況

  • Big-Omega（Ω）

    演算法時間函式的下限 Upper bound ➪ 最好狀況

  • Big-Theta（Θ）

    演算法時間函式的上限、下限

成長速率等級

$O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < … < O(n^k) < O(2^n) < … < O(k^n) < O(n!)$

空間複雜度 Space Complexity

執行演算法所消耗的記憶體，以 Big-O 呈現。

最佳解

滿足時間複雜度指標 → 降低空間複雜度

Example

迴圈需要執行 n 次 ➪ 時間複雜度＝O(n)

變數的數量為固定常數，共有 3 個（ x、cnt、i）➪ 空間複雜度＝O(1)

[DSA] 搜尋 Search

2022-06-25

分類

  • 內部搜尋

    資料量小，可從 RAM 資料檔中尋找資料。

  • 外部搜尋

    資料量較大，通常先分次載入部份資料，再用內部排序後回存或合併結果。

線性搜尋 Linear Search（= 循序搜尋 Sequential Search）

從頭逐一比對各筆資料值與目標，直到找到或搜完為止。

複雜度

  • 時間：O(1) → O(n) → O(n)

  • 空間：O(1)

程式碼

二分搜尋 Binary Search

在已排序陣列 中搜尋某一元素

過程

  1. 排序
  2. 找出中間值
  3. 比對目標與中間值（若大於目標往左找）
  4. 重複 3. 縮小範圍至找到或找完

複雜度

  • 時間：O(1) → O(logn) → O(logn)

  • 空間：O(1)

程式碼

內插搜尋 Interpolation Search

從已排序資料中以斜率預測目標所在位置

複雜度

  • 時間：O(1) → O(n) → O(log(logn))

  • 空間：O(1)

程式碼

指數搜尋 Exponential Search

搜尋無限、無邊界的已排序陣列

過程

  1. 從索引位置 $2^0$ 開始，不斷遞增 $2^1、2^2$ 直到 $2^i$。

  2. 若比目標大，會停止指數增長，改用二分搜尋來尋找。

複雜度

  • 時間：O(1) → O(logn) → O(logn)

  • 空間：O(1)

程式碼

深度優先搜尋 Depth-first Search（DFS）

過程

  1. 選定一起始點走訪
  2. 選與其相鄰且未走過的點，再以此為起始點。
  3. 重複 1～2.，直到走完所有頂點。

圖解

廣度優先搜尋 Breadth-first Search（BFS）

找出兩物之間的最短距離

過程

  1. 選定一起始點走訪
  2. 選與其相鄰且未走過的點
  3. 重複 1～2.
  4. 走完所有與其相鄰的點後，尋找下一層頂點。

分類

  • 內部排序 Internal sort（= 陣列排序）

相同值排序前後之順序

  • 簡單（初等）排序：排序演算法簡單，但執行時間較長 $O(n^2)$ ↔ 高等排序 $O(nlog^2n)$

堆積排序 Heap sort

步驟

  1. 陣列 → 最大堆積 Max Heap（遞增）或最小堆積 Min Heap（遞減）
  2. 再以完全二元樹來排序

動畫

程式碼

合併排序 Merge sort

採用分治法 Divide-and-Conquer，陣列拆分後再排序合併。

動畫

程式碼

快速排序 Quick sort

採用分治法 Divide-and-Conquer

步驟

  1. 選出一元素當基準 pivot

    左邊的元素都＜pivot、右邊的元素都 $\ge$ pivot（不用排序）

  2. 對左右邊的 array 重複 1.

動畫

程式碼

希爾排序 Shell sort（= 增量遞減排序 Diminishing Increment sort）

改良於插入排序

  • 將資料透過不同的間距 gap 拆分
  • 每段資料各自做插入排序
  • 利用每次排序出來的結果來加速下一次排序

動畫

程式碼

計數排序 Counting sort

先計算每個元素出現於序列中的次數，再用其進行排序。

動畫

程式碼

桶排序 Bucket sort

每一桶子對應資料區間，先將資料分配到不同桶，內部再各自排序。

速度

  • 當資料均勻分配每一桶時最快
  • 都分配同一桶中最慢

動畫

程式碼

基數排序 Radix sort

根據某個基數對元素進行分配排序

類型

  • Least Significant Digital（LSD）：由右到左
    1. LSD of each key：取得每一資料鍵值的最小位數（LSD）
    2. Sorting subroutine：依該位數大小排序
    3. Repeating：取得下一有效位數，並重複 2. 至最大位數（MSD）為止
  • Most Significant Digital（MSD）：由左到右

Client／Server 主從式架構

透過 HTTP 協定送出請求 request 給 server 提取 get，當 server 接收到回應 response 後開始處理，並回傳一個回應 response 給 client。

➪ client、server 之間透過 HTTP 協定進行溝通

  • 交互性強
  • 具有安全的存取模式
  • 響應速度快
  • 適於處理大量數據
  • 較難以維護、管理（成本高）
