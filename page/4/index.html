物件導向 Object-oriented

JS 是一種基於原型，而不是基於類的物件導向語言。

  • 採用物件 Objects 來模塑真實的實物世界
  • 每一物件都有自己的屬性與方法

基於類別 Class-Based
擁有「類別」與「實例」的概念

  • 類別 class：定義某種物件的 property
  • 實例 instance：類別產生的 object

ex. Java、C++

簡介

將變數、函式及 class 於創造階段時，先行指向記憶體位置 ➪ 往上提到函式頂端

影響

  • 不用先宣告變數，養成事後補上習慣 ➪ 可讀性降低

  • 互相呼叫函式

運作

執行函式後，但還沒開始跑內部程式碼之前依順序做出：

  1. 把參數放到 VO 並設定值，傳進什麼就是什麼，沒有值則設成 undefined。
  2. 把函式宣告放到 VO，若有同名則覆蓋掉。
  3. 把變數宣告（賦值不變）放到 VO，若有同名則忽略。

[JS] 執行環境 Execution Context

2022-03-04

執行環境 Execution Context（EC）

JS 的運作機制是一行行由上到下執行的單執行緒，當解析器碰到一段程式碼時，便會為其創造 EC。

類型

  • [全域執行環境 Global Execution Context（GEC）](##全域執行環境 GEC)
  • [函式執行環境 Function Execution Context（FEC）](##函式執行環境 FEC)
  • Eval 函式：把一串字串當作指令來執行

執行堆疊 Execution Stack

多個 EC 堆疊起來

  • 最上面的是正在執行中的程式
  • 一旦該函式執行完並且 返回 (return) 後，該 執行環境 就會自動從 執行堆疊 的最頂端消失。
  • 執行到所有的 執行環境 都完成任務、從堆疊中被移除
