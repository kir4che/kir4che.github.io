Scope 作用域

Global Scope

缺點

  • 允許重複宣告

Block Scope

於函式中的 { } 內宣告 ➪ 防止不當存取

宣告

var

in 函式作用域

let

in 區塊作用域

const

💡let、const 同區塊內禁止重複命名、不會洩漏到全域

jQuery 庫

JavaScript 函數庫，為事件處理特別設計的。

特性

  • HTML 元素選取、操作
  • CSS 操作
  • HTML 事件函數
  • JS 特效、動畫
  • HTML DOM 遍歷、修改
  • AJAX
  • Utilities

引用

  • 下載 jQuery
  • <script src="jquery.js"></script>

語法

$(選擇器).行為()

💡若同時用 jQuery 及其他框架，而都以 $ 作為簡寫怎麼辦？ noConflict()

特效

協助快速控制 CSS

  • show() 顯示 ↔ hide()
  • toggle() 切換隱藏／顯示
  • 淡入／出
    • fadeIn(speed)
    • fadeOut(speed)
    • fadeToggle(speed)
    • fadeTo(speed, opacity)
  • 滑入／出
    • slideDown(speed, callback)
    • slideUp(speed, callback)
    • slideToggle(speed, callback)
  • delay(speed) 延遲時間
  • stop() 停止正在發生的特效
  • animate(css, speed) 應用自定義動畫

Callback：於當前動畫完成後調用

Chaining：把多個動作／方法連接起来

HTML

内容＆屬性

  • text() 設置／回傳内容
  • html() 同上（包括 HTML 標記）
  • val() 設置／回傳表單值
  • attr() 設置／改變屬性值

新增内容

  • append() 結尾
  • prepend() 開頭
  • after() 元素後
  • before() 元素前

刪除内容

  • remove() 刪除包括其子元素
  • empty() 只刪除子元素

CSS

  • addClass() 新增一或多個類

  • removeClass() 刪除一或多個類

  • toggleClass() 切換新增／刪除

  • css() 設置／回傳元素之屬性

  • css() → style

遍歷

Browser Object Model（BOM）

瀏覽器本身（window 物件）所有功能的核心，作為 JS 與瀏覽器的溝通窗口，無關網頁內容。

結構

  • screen：使用者的螢幕畫面資訊
  • location：頁面網址 (URL)
  • history：瀏覽器的上下頁
  • navigator：瀏覽器資訊
  • frames：瀏覽器框架

常見操作

window

  • window.open() 開啟新視窗 ↔ window.close()

對話視窗

  • alert() 警告

  • confirm() 確認

  • prompt() 文字輸入

計時器

一段時間過後，自動執行某個函式 (callback)。

  • setTimeout() 執行一次就停止 ↔ clearTimeout()

  • ``setInterval()一直執行下去 ↔clearInterval()`

history

  • history.back() 回到上一頁 ↔ history.forward()
  • history.go() 跳至第幾頁

Document Object Model（DOM）

提供了一個文件（樹）的結構化表示法，定義程式可存取並改變架構、風格和內容的方法。

document 是 DOM tree 的根節點，表示整份 HTML 文件。

取得 DOM 元素

ID

  • document.getElementById()

Tag

  • document.getElementsByTagName()

Class

  • document.getElementsByClassName()

CSS 選擇器

  • document.querySelector()
  • document.querySelectorAll()

Node

  • node.childNodes() 元素的所有子節點
  • node.children() 元素的所有 Element 節點
  • node.firstChild() 第一個節點 ↔ lastChild()
  • node.previousSibling() 元素的前一個節點 ↔ nextSibling()

節點物件的屬性

  • node.nodeType() 取得該節點類型
  • element.innerHTML／Text() 取得 DOM 元素節點中的內容、改變元素的內容
  • element.outerHTML() 同上但會返回節點本身的內容
  • node.textContent() 第一個節點 ↔ lastChild()
    • textContent 返回 <script>、<style> 的所有內容，但 innerText 不會。
    • innerText 不會返回看不見的元素，但 textContent 會。

節點操作

  • document.createElement() 建立一新元素
  • document.createTextNode() 建立一新文字節點
  • parentNode.appendChild() 新增一新子元素到現有的最後面 ↔ removeChild()
  • parentNode.insertBefore() 新增一新子元素到現有的前面

[JS] AJAX 筆記

2022-06-25

AJAX 非同步的 JS 與 XML 技術

不必重整即可更新網頁

  1. 建立 XMLHttpRequest
  2. 傳送到伺服器要資料（常為 HTML、XML、JSON）
  3. 回傳到自己的瀏覽器
  4. 拿到後增加 onload 事件

XMLHttpRequest 跨瀏覽器撈資料

Ajax 核心，用於向伺服器交換資料。

var xhr = new XMLHttpRequest()

readyState

判斷目前讀取資料的狀態

  • 0 已產生 XMLHttpRequest，請求未初始化。

  • 1 設定連線方法，未傳送資料。

    xhr.open(method, url, async)

    • method
      • get 讀取：比 post 更簡單更快
      • post 傳送資料到伺服器：無大小限制、更安全
    • url
    • 異步 async
      • true 非同步：可同時進行多任務
      • false 同步：須等到請求完成才會繼續往下執行，常用於快速測試。

  • 2 偵測到正在使用 send

    xhr.send()

    💡若只要讀取資料則 () 內打 null

  • 3 資料處理中

  • 4 完成

onreadystatechange

調用於 readyState 發生變化時

onload

當資料全部跑完，才會觸發此事件

HTTP Status 狀態碼

判斷連線狀態

  • 1XX 參考資訊
  • 2XX 成功回應
    • 200 正確回傳
    • 201 資源已創建
  • 3XX 重新導向
  • 4XX 用戶端錯誤
    • 401、403 錯誤要求
    • 404 找不到
    • 429 請求次數過多
  • 5XX 後端錯誤
    • 500 不可預期之錯誤
    • 503 無法正確處理該請求

Cross-Origin Resource Sharing（CORS）跨來源資源共享

開啟資料權限才能讓其他開發者跨網域撈資料
