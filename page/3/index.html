[JS] 生成器 Generator

2022-03-25

Generator（ES6）

可隨時暫停、恢復執行的特殊函式

  • 封裝多個內部狀態的非同步

  • 像是狀態機 state machine，會一直改變內部的不同狀態。

流程

  1. 建立一生成器函式
  2. 呼叫 next() 開始執行函式
  3. 遇到 yield 暫停
  4. 再次呼叫 next() 從 yield 位置繼續往下執行

普通函式 vs 生成器

  • 宣告：普通函式 -> function、生成器 -> function*
  • 回傳值：普通函式 -> return、生成器 -> yield
  • 模式：普通函式 -> RTC、生成器 -> RPR
    • run to completion（RTC）：開始執行後，直到該函式所有程式碼執行完之前，不會執行别的程式碼。
    • run-pause-run（RPR）：函式執行中可被暫停一或多次，之後再恢復執行，且暫停期間允許執行其他程式碼

[JS] 陣列遍歷

2022-03-22

for

for (let i = 0; i < arr.length; i++)

arr.forEach

arr.forEach((value, index) => { ... })

for…in（ES5）

for (let i in arr)

  • 遍歷的是鍵 key
  • 常用來遍歷 object
  • 遍歷繼承的屬性

hasOwnProperty()

回傳物件是否有該屬性的布林值 ➪ 檢查是否為非繼承屬性

使用 for…in 要注意會遍歷包含繼承屬性的陣列，可能會產生預期外的結果。

for…of（ES6）

for (let v of arr)

  • 遍歷的是值 value
  • 不能遍歷 object
  • 可搭配 break、continue、return

forEach

於陣列中的每一元素上執行相同程式碼，但不更改陣列並回傳 undefined。

arr.forEach((value, index, array) => { ... })

  • 用法與 for loop 類似
  • 會改變原始陣列

map

陣列中的每一元素上執行相同程式碼，並回傳帶有更新元素的新陣列。

arr.map(value => { ... })

  • 不會改變原始陣列
  • 會回傳新陣列

filter

篩選陣列中滿足條件的元素，並回傳一新陣列，其中包含回傳條件的 truthy 元素。

arr.filter(value => { ... })

  • 不會改變原始陣列
  • 會回傳新陣列

every

判斷陣列中是否完全符合條件的元素，回傳 Boolean。

arr.every(value => { ... })

some

判斷陣列中是否部分符合條件的元素，回傳 Boolean。

arr.some(value => { ... })

reduce

將陣列中所有元素加總計算，回傳總和值。

arr.reduce((acc, value)=> acc + value)

  • callback 傳入參數
    • accumulator：目前累加值
    • currentValue：目前執行到的元素值
    • currentIndex：目前執行到的元素索引值
    • array：陣列本身
  • initialValue：初始累加值 ➪ 無指定則預設 0
