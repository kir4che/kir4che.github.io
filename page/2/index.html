[JS] Promise

2022-03-28

Promise（ES6）

非同步運算的最終完成或失敗的物件

  • 以建構式為原型，必須傳入一參數（該參數為函式）。

  • 透過微任務序列來驅動

    微任務序列 micro task queue

    事件循環機制 Event Loop 在 HTML5 Web API 標準中新增的

    觸發時機是在 Stack 清空時，JS 引擎會先確認微任務序列有無東西，有的話優先執行，直到清空後，才從 Queue 拿出新任務到 Stack 上。

優點

  • 非同步操作以同步化表達

    ➪ 易於閱讀、避免層層嵌套

  • 物件提供統一的接口

  • 物件狀態不受外界影響

  • 良好的錯誤處理

缺點

  • 一旦建立則立即執行，無法取消。
  • 若不設定回呼函式，Promise 內部錯誤不會反映到外部。
  • peding 狀態模糊，是剛開始還是快完成了。

[JS] 回呼函式 Callback

2022-03-25

簡介

一函式執行完再去執行另一函式（以參數型態傳入）

➪ 因工作切割而需要有 callback 來回頭處理之前未完成項目

優點

  • 確保函式時機與關聯
  • 可讀性高、易於維護

缺點

  • [回呼地獄 Callback Hell](##回呼地獄 Callback Hell)

    • 缺乏順序性
    • 程式碼之間黏著度高 ➪ 牽一髮而動全身
    • 無法使用 try…catch ➪ 無法除錯

  • [控制權轉移 Inversion of Control](##控制權轉移 Inversion of Control)
