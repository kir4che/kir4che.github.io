[JS] Web Storage

2022-04-27

簡介

可讓網頁將資料儲存於本地端的技術，資料以 key-value 形式保存。

優點

  • 儲存空間比 cookie 大很多（至少約 5MB）
  • 資料不會占用網路頻寬（與 cookie 相反）
  • 獨立的儲存空間 ➪ 避免混亂

缺點

  • 存於本地的資料未加密且永不過期 ➪ 隱私洩漏
  • 資料型別只能是 String
  • 隱私模式下不可讀
  • 爬蟲抓取不到
  • 無法跨域

類型

兩者差異主要在於壽命長短與有效範圍，而使用的 API 相同。

  • [Local Storage](##Local Storage)
  • [Session Storage](##Session Storage)

💡兩者皆限制在該網站的規範內，其他網站無法存取。。

Local Storage

本地永久性資料，除非顯式刪除或清空，且資料可跨瀏覽器分頁（生命週期長）。

方法

  • 新增：localStorage.setItem(key,value)
  • 讀取：localStorage.getItem(key)
  • 刪除
    • 單個：localStorage.removeItem(key)
    • 所有：localStorage.clear()
  • 查找：localStorage.key(index)

Session Storage

關閉瀏覽器視窗或分頁關閉就沒了（生命週期短）

方法

  • 新增：sessionStorage.setItem(key,value)
  • 讀取：sessionStorage.getItem(key)
  • 刪除
    • 單個sessionStorage.removeItem(key)
    • 所有：sessionStorage.clear()
  • 查找：sessionStorage.key(index)
