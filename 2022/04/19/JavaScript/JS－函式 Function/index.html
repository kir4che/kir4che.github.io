[JS] 函式 Function

2022-04-19

將一或多段程式指令包裝起來，可重複使用，也方便維護。

宣告

宣告式

function Name([param]) { ... }

表達式

  • 具名函式 Named Function

    var Name1 = function Name2([param]) { ... }

  • 匿名函式 Anonymous Function：沒有方法名的方法

    var Name = function([param]) { ... }

    • 透過呼叫變數來執行函式內容
    • 用於容易解決或一次性使用

建構式（效能差）

const Name = new Function ([param], ...)

箭頭 Arrow（ES6）

const Name = ([param]) => { ... }

  • 匿名函式的一種，隱式回傳值。

  • 無法當作構造子 constructor

  • 沒有引數：若在箭頭函式呼叫引數，報錯：arguments is not defined。

  • 没有自己的 this

    • 傳統：呼叫方法的物件
    • 箭頭：該函式外層作用域

  • 無法以 bind()、apply()、call() 重新綁定 this

  • 沒有原型 prototype 屬性

回傳 return

函式執行後 return 會中斷程式並回傳一值

  • 可接受多個參數
  • 只能回傳一值
  • 若沒有值回傳，則回傳 undefined。

參數 Parameters

是在函式的那些傳入參數名稱的定義

預設參數 Default Parameters（ES6）

沒有值傳入的情況下，參數能以指定預設值初始化。

  • 減少程式碼
  • 利於代碼的可讀性

引數 Arguments

當函式被呼叫時，傳入到函式中真正的值。

  • 函式一創建就有
  • 呼叫方法來取得所有傳入的參數，長度也以其決定。
  • 是一個類陣列 Array-Like（不是真正的陣列）
  • 記憶體空間獨立
  • 若傳入的參數＜命名參數，命名參數與不會同步。

類陣列 Array-like

一樣有 length、索引從 0 開始等特性，但沒有陣列的內建方法。

ex. forEach()、map()

需要用到嗎？

不要！此有設計缺陷，新手使用容易出問題，可改用其餘參數 rest。

this

指向當前物件

  • 永遠指向一個物件
  • 指向取決於函式調用的位置

規則（以優先排序）

  • new 綁定：指向 new 出來的物件

  • 明確綁定（call、apply、bind）：明確指向要綁定給 this 的物件

  • 隱含綁定：當函式為物件的方法時，在執行階段 this 就會被綁定至該物件。

  • 預設綁定：以上規則都不適用時套用

    • 非嚴格模式：this -> window
    • 嚴格模式下：this -> undefined。

  • 沒有明確呼叫者 -> window

綁定

  • call()

  • apply()

  • bind()：強制綁定，綁定後就無法更改。

IIFE

全名：Immediately Invoked Function Expression，立即呼叫函式表達式。

  • 全域變數減少
  • 全域變數區域化
