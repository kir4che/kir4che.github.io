[JS] AJAX

2022-04-26

Asynchronous JavaScript and XML（AJAX）

網頁前端與後端伺服器溝通的技術

  • 核心：XMLHttpRequest（XHR） 物件
  • 目的：不必重整網頁即可向伺服器傳送及取得資料
  • 流程：所有 AJAX 行為皆由網頁端先發出，透過 API 接口對後端進行請求，而後端會針對請求的內容進行回應。

API 接口

所有 AJAX 都有對應的 API 接口，其實就是一段網址，不同網址也對應不同「HTTP 請求方法」，請求方法必須與網址完全對應才可運作。

類似於點餐的櫃台

網頁端請求

AJAX 會統一由網頁端發出請求，依據不同接口、請求方法進行請求。

類似於點餐的顧客，且說明需要什麼餐點…。

伺服器回應

收到請求後，伺服器會依據請求的方法、內容與資料庫取得資源並統整過濾後進行回應。

類似於櫃台員工對於顧客的回應

HTTP 請求方法

GET

請求特定的資源（接在 url 後），且不會提交任何資源。

➪ 伺服器僅會依據條件進行回應

POST

提交特定的資源（待在請求體中），常用於新增資源或改變狀態（登入、註冊…）

➪ 伺服器會回應新增或改變狀態的結果

GET vs POST

  • GET：比 post 更簡單更快
  • POST：無資料量限制、更安全

一定要使用 POST 情況

  • 無法使用緩存檔案
  • 向伺服器傳送大量資料

PUT

取代指定資源，常用於完整更新 ➪ 伺服器會回應更新的結果

PATCH

更新指定資源，常用於部分資源更新 ➪ 伺服器會回應更新的結果

DELETE

刪除特定的資源 ➪ 伺服器會回應刪除的結果

XMLHttpRequest 跨瀏覽器撈資料

var xhr = new XMLHttpRequest()

readyState

判斷目前讀取資料的狀態，回傳 unsigned short。

  • 0 已產生 XMLHttpRequest，請求未初始化。

  • 1 建立連接（未傳送資料）

    xhr.open(method, url, async, [user, password])

    • method：GET、POST
    • 非同步 async：true 非同步（預設）、false 同步

  • 2 傳送請求

    xhr.send(body)

    💡若只要讀取資料則 () 內打 null

  • 3 資料處理中

  • 4 完成

onreadystatechange

呼叫於 readyState 發生變化

onload

當資料全部跑完，才會觸發此事件

HTTP Status 狀態碼

優先確認 AJAX 成功與否的方式，透過數字進行不同的分類，讓開發者可以優先專注在可能出錯的環節。

1XX 參考資訊

2XX 成功回應

  • 200 正確回傳
  • 201 資源已創建

3XX 重新導向（轉址）

4XX 用戶端錯誤

  • 401、403 未取得授權
  • 404 找不到資源
  • 422 不符合操作流程
  • 429 請求次數過多

5XX 伺服器端錯誤

  • 500 不可預期之錯誤
  • 503 無法正確處理該請求
