[JS] 錯誤處理

2022-04-18

當錯誤 error（例外 exception）發生時，會產生一個用來代表錯誤的物件。

類型

Error

一般的錯誤情況，常用於客製化例外情況。

RangeError

表示數值或參數超出有效範圍

ReferenceError

表示存取一個不存在的變數，常發生於拼錯字的情況。

SyntaxError

表示程式碼違反 JS 的語法規則，使得解析程式碼時發生錯誤。

TypeError

表示值的型別非預期操作時發生之錯誤，常發生於呼叫執行一不存在的函式時。

URIError

表示給 encodeURI()、decodeURl() 傳遞的 URI 無效，但兩者容錯性高，不易發生此錯誤。

EvalError

表示 eval() 使用錯誤，已不被當前 ECMAScript 規範所採用。

處理錯誤 Handling Exceptions

try...catch...finally

try

於 try 區塊內試著執行程式碼

catch

一有任何錯誤發生時，則立即跳到 catch，沒有則跳過 ➪ 阻止錯誤向外冒泡

  • catchID：catch 中可指定的一個參數，離開則無法取得。

finally

最後執行 fially

  • 用於包含清除的程式碼（ex. closing files）
  • 若 finally 中有回傳值，其為整個 try...catch...finally 回傳結果。

客製化錯誤 Throwing Exceptions

JS 允許開發者透過 throw 拋出幾乎任何型別的錯誤（仍建議用 JS 內建）

客製化錯誤訊息

ex. 透過除法卻不小心除了分母為 0 的數值
