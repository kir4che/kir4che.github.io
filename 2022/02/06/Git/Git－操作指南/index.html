[Git] 操作指南

2022-02-06

簡介

分散式版本控制工具，運作不需伺服器端軟體

用處

  • 版本備份
  • 切換版本
  • 協同開發

分區

  • 工作區 working directory：實際操作的資料夾（檔案）
  • 暫存區 stage snapshot（= .git）：放置工作區未提交之檔案
  • git 目錄 .git directory：歷史紀錄

終端機指令

cmd wins mac
切換目錄 cd cd
取得目前位置 cd pwd
列出目前的檔案列表 dir ls
建立新目錄 mkdir mkdir
建立檔案 X touch
複製檔案 copy cp
移動檔案 move mv
刪除檔案 del rm
清除畫面上的內容 cls clear

建立專案

本地數據庫 Local Repository

  • 初始化 git init

  • 當前狀態 git status：哪些項目被修改過

  • 修改内容 git diff

提交檔案

  1. 加入索引（追蹤）

    • 全部 git add .

    • 指定 git add 檔名

  2. 提交 git commit -m 訊息(自行撰寫)

    • 改寫 git commit --amend

版本管理

歷史資訊

  • 過去 git log
  • 未來 git reflog

切換

  • 指定版本 git checkout 識別碼（前4碼）
  • 切回最新版本 git checkout master

HEAD （= HEAD～0＝＠）當前版本：會隨著 master 推進

  • HEAD^＝HEAD~1 上一版本
  • HEAD^^＝HEAD^2 上上一版本
  • HEAD^^^＝HEAD^3 上上上一版本

還原

清除過去紀錄

  • 還原就對了（預設 –mixed） git reset 分支名
  • 保留工作區、暫存區的檔案 git reset --soft 分支名
  • 放棄工作區、暫存區的檔案 git reset --hard 分支名

檔案管理

取消索引

  • 放進.gitignore
  • 刪除檔案 git rm＝git add -u＝git add --update

搬移

git mv 檔名 資料夾

重命名

git mv 舊檔名 新檔名

還原

  • 清空工作區（未加入索引）

    • 顯示要被清除之檔案 git clean -n
    • 強制清除 git clean -f

  • 還原工作區（已加入索引）

    • 全部 git checkout -- 檔名
    • 單一 git checkout .

  • 退到工作區（已加入索引） git reset HEAD

暫存

  • 至當前目錄git stash

  • 瀏覽 stash 列表 git stash list

  • 還原最新暫存 git stash pop

  • 清除

    • 最新 git stash drop

    • 全部 git stash clear

上傳至 Github（遠端數據庫）

加入遠端庫

  • 連接 git remote add 遠端庫（預設 origin） 專案網址

    • 重命名 git remote rename 原遠端庫名 新遠端庫名

  • 推送 git push 遠端庫 分支

    • 強制 git push -f

克隆

git clone 遠端庫網址

使用時機

  • 本地數據庫資料不見，從遠端庫拉一份新資料。
  • 推送至 GitHub，其他開發者想拉下來開發時。

下載

git pull 遠端庫＝git fetch＋git merge

分支 branch

分開儲存各個修改記錄，互相不受其他分支的影響，最初提交的預設為 Master。

類型

  • Integration ➪ 隨時建立發布版本
  • Topic ➪ 開發功能或修復錯誤…

建立

git branch 分支名

  • 刪除 git branch -D 分支名
  • 重命名 git branch -m 舊分支名 新分支名

合併

完成任務的 topic（歷史紀錄），最終都會併到 integration。

  • 類型

    • Merge

      git merge 分支名

      修改內容的記錄維持原狀，但合併後會變更複雜。

    • Rebase

      git rebase 分支名

      修改內容的記錄接在要合併的分支後，合併後會較清楚，易發生衝突。

  • 取消合併

  • 解決衝突

標籤

紀錄訊息

新增

  • 輕量標籤 git tag 標籤名
  • 標示標籤 git tag -am "備註" 標籤名
  • 於特定 commit git tag 標籤名 識別碼

刪除

git tag -d 標籤名

列出現有標籤

git tag

切換至標籤的 commit

git checkout 標籤名

查詢詳細內容

git tag -n
