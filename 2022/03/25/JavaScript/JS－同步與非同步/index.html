[JS] 同步與非同步 Sync＆Async

2022-03-25

同步與非同步

同步 Synchronous

一次只能做一件事

  • 阻塞後面程式碼的執行
  • 瀏覽器同步執行時，阻塞畫面的渲染。

非同步 Asynchronous

一次同時做好幾件事

單執行緒的 JS 為什麼可以非同步？

JS 只有一個主執行緒，還有其他執行緒，當然所有函式任務只執行於主執行緒。

  • 瀏覽器內核除了 JS 引擎執行緒，還有其他同步執行的執行緒。
  • setTimeout、event、Ajax… 無法預期執行時間的操作，都會先被丟到事件佇列 Event Queue 排隊，等候執行，等到同步執行的程式碼執行完，才會處理那些佇列中的任務。

非同步解決發展歷程

回呼函式 Callback Function

一函式執行完再去執行另一函式

問題：回呼函式僅能配合 try…catch 捕捉錯誤，當非同步錯誤發生時難以控制，也可能造成回呼地獄 Callback Hell。

Promise（ES6）

非同步運算的最終完成或失敗的物件

問題：在熟悉度不足的開發者手上，只是從 Callback Hell 變成 Promise Hell。

生成器 Generator（ES6）

可隨時暫停、恢復執行的特殊函式

Async / Await（ES8）

將非同步事件用同步語法來處理

問題：多個非同步程式碼沒有依賴性卻用 await ➪ 性能降低
