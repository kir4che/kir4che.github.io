    Null object
    Boolean boolean
    Number number
    String string
    Function 物件 function
    所有其它物件 object

Truthy & Falsy

True

  • 'Hello World'：非空字串
  • 20：非零的有效數字
  • [], [1]：陣列（不管是不是空的）
  • {}, { name: 'Molly' } 物件（不管是不是空的）
  • function func() {} 函式
  • true

False

  • ""：空字串
  • 0, -0, NaN
  • null
  • undefined
  • false

null vs undefined

上述結果發現兩者相等，但做嚴格相等時，兩者又不同。

相同處

  • Boolean 轉換皆是 false

不同處

  • null 是有效 JSON 值，undefined 不是。
  • typeof 轉換 null -> Object，undefined -> undefined。
  • Number 轉換 null -> 0，undefined -> NaN。

為什麼 null 是 Object？

JS 一開發就是如此，至今無法修正，且會導致不可預期的錯誤。

undefined vs undeclared

未定義 undefined

作用域中宣告過但没有賦值的變數，存取不會報錯且回傳 undefined。

未宣告 undeclared

沒有在作用域中宣告過的變數，存取會報錯且執行中斷。
