[TS] Class＆Interface

2022-06-25

Class 類別

OOP

  • 封裝 public、private、protected

  • 繼承 extend

    💡不可多重繼承，但可透過 Mixins 或 Implements

  • 多型：繼承後產生不同的類別，對同一方法

靜態方法

不可被實例繼承，僅存在類別。

accessors 存取器

有效間接操作類別中的屬性 ➪ 提高屬性安全性

  • getter

    存取子讀取屬性值，不能有參數，且一定要有回傳值

  • setter

    存取子修改屬性值，只能有一參數。

abstract 抽象

使類別不可實例化（new），但其方法必須被子類別實現。

implements 實現

指定要實作之介面，使其行為具體化。

類別型別

Interface 介面

約束類別的行為，只描述屬性、方法。

屬性

  • 可選屬性 name?: string

  • 唯獨屬性 readonly name?: string：賦值後即不可變，類似於 const。

  • 任意屬性 [name: string: any]

    💡一旦定義任意屬性，其他所有屬性都必須為其子屬性。
