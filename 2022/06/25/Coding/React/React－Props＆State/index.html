[React] Props＆State

2022-06-25

屬性 Props

父組件 → 子組件傳遞之屬性，用來設定組件初始化的相關資料，可讀不可寫。

PropTypes

為組件參數作類型限制

children

JSX element 的子 element，調用包在標籤中的內容。

  • React.FC

    已在 FC 定義好 children 型別、函式回傳值

    💡不可使用 setState()，須改用 useState()、useEffect() 等。

  • 直接定義

    • JSX.Element（不支持 string）
    • ReactChild
    • ReactNode

狀態 State

維護組件內的資料，用於會隨時間而變動之事件，為私有屬性。

初始化

於 constructor 定義初始 state

💡JS this 取決於當下誰執行 function

setState

setState(updater, [callback])

改變組件 state，並重新調用 render 以渲染。

  • 接受 object、function 作為參數
  • 不會馬上更新，須利用利用上一次結果 ➪ prevState

💡不要寫在 render() ➪ 死迴圈

生命週期

Mounting

建立組件時被執行

componentDidMount() 於組件被 mounted 後執行 ➪ 啟動

Updating

更新組件時執行

componentDidUpdate() 於 re-rending 更新組件後執行

Unmounting

當組件將要從 DOM 刪除時觸發

componentWillUnmount() 於組件被刪除前執行 ➪ 清除

Function Steps

  1. 定義子組件之 Props type

  2. 調用 useState

  3. 設定 setState

  4. 子組件功能偵測父組件中的 function

  5. 調用 function 執行 setState 來更新 state

  6. 觸發渲染機制

  7. 重新渲染 App 所有組件

Class Steps

  1. 定義父、子組件之 Props、State type

  2. 父、子組件 constructor 初始化

  3. super(props)

  4. this.state

  5. 調用 setState

  6. 記得保證 this 永遠指向 APP，不管是誰執行它。

    this.function = this.function.bind(this)
