[DSA] 時間、空間複雜度

2022-06-24

演算法 Algorithm

解決特定問題或提供某功用之有限指令／敘述之集合

  1. Input：≥ 0 個 input data

  2. Output：至少 1 個 output 結果

  3. Definiteness：每個指令 / 敘述皆 clear, unambiguous

  4. Finitense：結束於有限個步驟

    ▴Program 不一定有限

  5. Effectiveness：步驟合理、可執行

時間複雜度 Time Complexity

執行演算法所需時間，以 Big-O 呈現。

漸進符號 Asymptotic Notation

  • Big-O（Ο）

    演算法時間函式的上限 Upper bound ➪ 最壞狀況

  • Big-Omega（Ω）

    演算法時間函式的下限 Upper bound ➪ 最好狀況

  • Big-Theta（Θ）

    演算法時間函式的上限、下限

成長速率等級

$O(1) < O(logn) < O(n) < O(nlogn) < O(n^2) < … < O(n^k) < O(2^n) < … < O(k^n) < O(n!)$

空間複雜度 Space Complexity

執行演算法所消耗的記憶體，以 Big-O 呈現。

最佳解

滿足時間複雜度指標 → 降低空間複雜度

Example

迴圈需要執行 n 次 ➪ 時間複雜度＝O(n)

變數的數量為固定常數，共有 3 個（ x、cnt、i）➪ 空間複雜度＝O(1)
