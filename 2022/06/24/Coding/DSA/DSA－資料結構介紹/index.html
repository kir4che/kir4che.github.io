[DSA] 資料結構介紹

2022-06-24

陣列 Array

特性

  • 儲存有序串列之相同型態的資料於連續記憶空間 ➪ 新增、刪除慢
  • 須事先宣告固定的記憶體空間 ➪ 存取快 O(1)

串列 List

元素以線性方式（有序）儲存，且可重復。

類型

  • [**Linked list**](##鏈結串列 Linked list)（無序）
  • ArrayList：自動擴張容量

鏈結串列 Linked list

特性

  • 不用事先宣告連續的記憶體空間，資料數量為動態 ➪ 新增、刪除快
  • 存取 O(n) 節點 node 須從頭開始找 ➪ 存取慢

類型

  • 單向鏈結串列 Singly Linked List

    追加資料只須更動指標的指向

  • 雙向鏈結串列 Doubly Linked List

    追加或刪除資料時，須更動比單向鏈結串列更多的指標次數。

  • 環狀鏈結串列 Circular Linked List

    每個節點都可以是串列頭

用途

  • 無法預知、需頻繁變動資料的數量
  • 大量新增、刪除資料

集合 Set

特性

  • 無序、元素不重複

類型

  • HashSet（高效） ➪ 快速查找
  • LinkedHashSet（有序）➪ 快速查找
  • TreeSet（有序）➪ 確保元素處於有序

用途

  • 一堆可能重複的資料
  • 找出共通之元件

Map

鍵映射至值的集合

特性

  • 每一映射的鍵不重複
  • 每一鍵最多只能映射一值

類型

  • [**HashTable**](##雜湊表 Hash Table)：以雜湊函式 Hash Function 算出鍵值並放入對應的陣列索引，且不允許鍵值為 null。

  • HashMap（高效）：允許鍵值為 null

    • 新增 put
    • 刪除 remove()
    • 取得 get()
    • 查找指定鍵值 containsKey()、containsValue()

  • LinkedHashMap（有序）

  • TreeMap（有序）

堆疊 Stack

特性

  • 先進後出 FILO、後進先出 LIFO
  • 只能從頂端存取資料
  • 只需一個指標：top

實作

  • Array

  • Linked list

常見操作

  • push() 新增元素 O(1)
  • pop() 刪除頂端元素 O(1)
  • peek() 查看頂端元素

用途

  • 上一頁
  • 還原

佇列 Queue

特性

  • 先進先出 FIFO的有序表

  • 不易於插入或刪除某指定資料

  • 若要取出最新資料，需先移除前面所有資料。

  • 前端 front、尾端 rear（back）

特殊類型

  • Circular Queue：開頭結尾接起來 ➪ 解決 front 前面有未使用空間

    當 rear 到達 MAXSIZE 時，將其設為 0。

  • Deque（Double-ended queue、Head-tail linked list）：兩端都可放資料、拿資料

    • Input restricted deque：只有一邊可放入 ↔ Output restricted deque

  • Priority Queue：看誰有優先權 ➪ 不符合 FIFO　ex. 工作排程

  • Multiple Queue：兩個 queue 共用一 array

樹 Tree

名詞介紹

  • 節點 Node：每一資料元素
  • 枝 Branch：節點向下延伸擴展的枝
  • 根節點 Root：樹狀結構的最上層，具有唯一性。
  • 葉節點 Leaf：沒有子節點的節點 ↔ 非終端點 Non-terminal Nodes
  • 祖先節點 Ancenstors：某節點以上的節點 ↔ 子孫節點 Descendant
  • 父節點 Parent、子節點 Child：父節點向下分支出子節點
  • 兄弟節點 Siblings：同一父節點的其他節點
  • 分支度 Dregree：每一節點的節點數量，且每條分支可看作 Linked list。
  • 階層 Level：節點的層級數量（根的階層＝ 1）
  • 高度Height：某節點下最長邊之距離
  • 深度 Depth：根節點至某節點之距離（根的深度＝ 0）

特性

  • 非線性結構
  • 不能形成環路 ➪ 爸爸只有一位

二元樹 Binary tree

每一節點的子節點 ≤ 2，且分為左右子樹。

類型

  • 歪斜樹 Skewed binary tree：分支只有左或只有右

  • 嚴格二元樹 Strictly binary tree：每一節點的子節點只能為 0 或 2

  • 滿二元樹 Full binary tree（= Perfect binary tree）

    若階層為 k，節點數為 $2^k-1$。

  • 完全二元樹 Complete binary tree：最後一層以外皆全滿

    若階層為 k，$2^{(k-1)} \leq$ 節點數 $\leq 2^k-1$

  • 二元搜尋樹 Binary Search tree（= 有序二元樹）

    • 左子樹鍵值 < 樹根鍵值 < 右子樹鍵值

    • 左右子樹亦是二元搜尋樹

    • 每一鍵值皆不相同

    • AVL（Adelson-Velsky and Landis）樹

      每一節點的左右邊子節點高度差＝ -1～1 ➪ 搜尋更快

    • 紅黑樹 Red–Black tree ➪ 保持樹狀結構的平衡

      • 每一節點是🔴或⚫，且根為⚫。
      • 葉節點一定是⚫的 NULL
      • 🔴的子節點必為⚫ ➪ 不能🔴—🔴，但⚫的子節點無限制。
      • 從任何節點出發，其下至葉節點所有路徑的⚫數目要相同。

遍歷 Traversal

  • Pre-order 前序：根 → 左 → 右 ➪ 深度優先搜尋 Depth-First Search（DFS）
  • In-order 中序：左 → 根 → 右
  • Post-order 後序：左 → 右→ 根
  • Level-order 層序：一層層從左至右 ➪ 廣度優先搜尋 Breadth-First Search（BFS）

堆積 Heap

一種完全二元樹

類型

  • 最小堆積 min-Heap
    • 所有父節點的值 < 子節點
    • 根＝min
  • 最大堆積 Max-Heap
    • 所有父節點的值 > 子節點
    • 根＝MAX

雜湊表 Hash Table

名詞介紹

  • 雜湊函式 Hash Function：將資料打亂並混合，建立出該資料對應的雜湊值。
  • 桶 bucket：儲存資料的位置，每一位置對應唯一位址 bucket address。
  • 槽 slot：每一桶中的資料欄位
  • 碰撞 collision：對應到相同位址 ➪ 以連結串列的方式儲存
  • 溢位 overflow：資料經過雜湊函數運算後，雜湊值所指向的桶位址已滿，無法再儲存。

原則

  • 降低碰撞與溢位的產生
  • 不宜過於複雜，越容易計算越佳
  • 文字 → 數字的鍵值
  • 雜湊值均勻分布在每一桶 ➪ 均衡性

雜湊函數

  • 直接定址法

    以鍵值 k 或 k＋常數 c 作為雜湊值

  • 數字分析法

    提取鍵值中取值比較均勻的數字作為雜湊值

  • 除留餘數法

    將鍵字 ÷ 某個＜哈希表長度 m，所得餘數為雜湊值。

  • 分段疊加法

    依雜湊表地址位數，將關鍵字分成位數相等的幾部分並相加，雜湊值為捨棄最高位後的總和。

  • 平分取中法

    當鍵值部分分佈不均勻時，可求出其平方值，然後依需求取中間幾位作為雜湊值。

  • 偽隨機數法

    採用一偽隨機數當作雜湊函數

處理碰撞、溢位

  • 線性探測法 Linear Probing

    以線性方式往後找到空 bucket 為止，若後面 bucket 滿了，可循環到前面找。

  • 平方探測法 Quadratic Probing

    $(H(x)±i^2)\ mod \ b\ \ _{1 ≤ i ≤ \frac{(b-1)}{2}}$ ➪ 尋找其他空 bucket

  • 鏈結法 Chaining

    於每一 Bucket 中用 Linked List

  • 再雜湊法 Rehashing

    用其他雜湊函式計算另一雜湊函數地址，重覆計算到不再衝突。

用途

  • 關鍵字搜尋
  • 加／解密

圖 Graph

由數個頂點 vertex（節點 node）、邊 edge 所構成，且之間互相連結或關聯 ➪ $G=(V,E)$

名詞

  • 路徑 path：兩頂點經過的邊

  • 路徑長度 path length：該路徑上所有邊的數量

  • 簡單路徑 simple path：起點～終點的頂點不可重複（起點可＝終點）

  • 相鄰 adjacent : 兩頂點有一條邊相連

  • 分支度 degree

    • 有向圖中：入分支度 In-Degree ↔ 出分支度 Out-Degree
    • 無向圖中：一個頂點含有幾條邊

  • 完整圖 complete graph

    • 有向圖中：n 個頂點，會有 $n(n-1)$ 條邊
    • 無向圖中：n 個頂點，會有 $\frac{n(n-1)}{2}$ 條邊

  • 子圖 subgraph

類型

  • 有向圖 Directed graph：邊具有方向性 ➪ 單向

  • 無向圖 Undirected graph ➪ 雙向

  • 權重圖 Weighted graph：數值說明連接程度

表示法

  • 相鄰矩陣 Adjacency Matrix：兩頂點間有邊值為 1，無為 0。

  • 相鄰串列 Adjacency List：每一頂點以 Singly Linked List 串接相鄰頂點，無邊不處理。
