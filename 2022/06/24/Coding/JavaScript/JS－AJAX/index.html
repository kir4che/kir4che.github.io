AJAX 非同步的 JS 與 XML 技術

不必重整即可更新網頁

  1. 建立 XMLHttpRequest
  2. 傳送到伺服器要資料（常為 HTML、XML、JSON）
  3. 回傳到自己的瀏覽器
  4. 拿到後增加 onload 事件

XMLHttpRequest 跨瀏覽器撈資料

Ajax 核心，用於向伺服器交換資料。

var xhr = new XMLHttpRequest()

readyState

判斷目前讀取資料的狀態

  • 0 已產生 XMLHttpRequest，請求未初始化。

  • 1 設定連線方法，未傳送資料。

    xhr.open(method, url, async)

    • method
      • get 讀取：比 post 更簡單更快
      • post 傳送資料到伺服器：無大小限制、更安全
    • url
    • 異步 async
      • true 非同步：可同時進行多任務
      • false 同步：須等到請求完成才會繼續往下執行，常用於快速測試。

  • 2 偵測到正在使用 send

    xhr.send()

    💡若只要讀取資料則 () 內打 null

  • 3 資料處理中

  • 4 完成

onreadystatechange

調用於 readyState 發生變化時

onload

當資料全部跑完，才會觸發此事件

HTTP Status 狀態碼

判斷連線狀態

  • 1XX 參考資訊
  • 2XX 成功回應
    • 200 正確回傳
    • 201 資源已創建
  • 3XX 重新導向
  • 4XX 用戶端錯誤
    • 401、403 錯誤要求
    • 404 找不到
    • 429 請求次數過多
  • 5XX 後端錯誤
    • 500 不可預期之錯誤
    • 503 無法正確處理該請求

Cross-Origin Resource Sharing（CORS）跨來源資源共享

開啟資料權限才能讓其他開發者跨網域撈資料
