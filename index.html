[JS] JavaScript 是什麼？

2022-07-01

一種物件導向的用戶端腳本語言，能夠動態更新網頁內容、控制多媒體、動畫…。

  • 解釋性：原始程式碼不需編譯，直接在瀏覽器上運行時被解釋。
  • 基於原型的面向物件：能運用自己已創建的物件
  • 事件驅動：在主頁執行某種操作所產生的動作。JS 可直接對用戶端做出回應，並以事件驅動進行，無需經過 Web 服務程式。
  • 跨平台：JS 依賴於瀏覽器本身，與操作環境無關。
  • 安全性：不允許訪問本地磁碟，且不能將數據存入伺服器上，也不允許修改、刪除網路文本，有效防止數據丟失。

[JS] Async / Await

2022-06-20

Async / Await（ES8）

將非同步事件用同步語法來處理

  • Promise、Generator Function 尬在一起的語法糖
  • async 與 await 成對出現

優點

  • 同步寫法、非同步操作 ➪ 可讀性提高
  • 條件語句與其他流程語句可直接寫到判斷條件
  • 處理複雜流程時，清晰度有優勢。

優缺點

  • 無法處理 Promise 回傳的 reject，要用 try…catch 捕獲。

  • 做不到並行（Promise✅）

    也不是不能，只要 await 不在同一 async 函式中即可並行。

  • 全域捕獲錯誤須用 window.onerror（Promise 有專用的）

    window.onerror：捕獲各種稀奇古怪的錯誤，造成系統浪費。

缺點

  • try…catch 內部變數無法傳遞給下一 try…catch
  • 無法實現 Promise 原生方法　ex. .race()

以下時機請用 Promise

  • 要用 Promise 各種原生方法
  • 需要並行
  • 傳遞參數
  • 以 .catch（） 處理錯誤

用法

async

宣告一非同步函式，關鍵字放 function 前面。

  • 一種物件種類
  • 跑完會回傳 Promise（resolved or rejected），可接 then 繼續連結。
  • 函式内部 return 回傳值會成為 then 回呼函式的參數

await

呼叫並等待該非同步函式，關鍵字放呼叫 async function 前面。

  • 屬於一元運算子
  • 多個 await 後面的非同步操作，若是互不依賴，最好讓它們同時觸發。

try…catch

錯誤處理：運行正確則往下執行，失敗則跳到 catch。

不在 async 就無法用 await，所以常用 .then/catch() 處理結果或 Error。

由來：JS 遇到 0、null、undefined，會自動轉型成 false，但有時 0 是正常值，只能容錯到 undefined、null。

提供一個標準方式去獲取不同執行環境下的全域物件

[JS] Web Storage

2022-04-27

簡介

可讓網頁將資料儲存於本地端的技術，資料以 key-value 形式保存。

優點

  • 儲存空間比 cookie 大很多（至少約 5MB）
  • 資料不會占用網路頻寬（與 cookie 相反）
  • 獨立的儲存空間 ➪ 避免混亂

缺點

  • 存於本地的資料未加密且永不過期 ➪ 隱私洩漏
  • 資料型別只能是 String
  • 隱私模式下不可讀
  • 爬蟲抓取不到
  • 無法跨域

類型

兩者差異主要在於壽命長短與有效範圍，而使用的 API 相同。

  • [Local Storage](##Local Storage)
  • [Session Storage](##Session Storage)

💡兩者皆限制在該網站的規範內，其他網站無法存取。。
