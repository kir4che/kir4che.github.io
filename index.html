指定型別

Type Inference 型別推斷

在沒有明確指定型別時推測出型別，編譯時會報錯。

＜型別＞= 值

值 as 型別

💡並非轉換，不可斷言成聯合型別中不存在的型別。

基本型別

原始資料型別

string

number

boolean

空值

  • null：賦予空值

  • undefined：宣告卻未被賦值

物件資料型別

object

💡注意

arrays

類別陣列 ≠ arrays

有自己的介面定義　ex. IArguments, NodeList, HTMLCollection

function

一個函式有輸入和輸出，可以針對參數(輸入)及返回值(輸出)進行型別規範。

  • 型別註釋

  • 定義

class 類別

void

無回傳值的函式

擴充型別

tuple 元組

tuple 就是合併了不同型別的物件。

enum 列舉

管理一系列的常數，作為狀態判斷用。

interface 介面

Literal Type 字面值型別

某些特殊的「值」可當作「型別」使用。

  • string literal types

  • numeric literal types

  • non-literal types　ex. boolean、enum、interface, etc.

特殊型別

  • any 任意值

    可被賦值為任意型別，宣告時未指定型別也是 any。

  • unknow

    接受任何形態賦值，只能賦值給自己或 any。

  • never

    不應該存在的型別，用於錯誤處理函式，且需回傳，但因錯誤導致不會有回傳值。

複合型別

  • union types 聯合型別

    以｜取多種型別的其一 ≒ OR

    Type Alias 型別別名

    給一型別起個新名字

  • intersection types 交集型別

    以＆取多種型別的其一 ≒ AND

Generic 泛型

定義函式、介面或類別時先不指定型別，使用時再指定。

[TS] Class＆Interface

2022-06-25

Class 類別

OOP

  • 封裝 public、private、protected

  • 繼承 extend

    💡不可多重繼承，但可透過 Mixins 或 Implements

  • 多型：繼承後產生不同的類別，對同一方法

靜態方法

不可被實例繼承，僅存在類別。

accessors 存取器

有效間接操作類別中的屬性 ➪ 提高屬性安全性

  • getter

    存取子讀取屬性值，不能有參數，且一定要有回傳值

  • setter

    存取子修改屬性值，只能有一參數。

abstract 抽象

使類別不可實例化（new），但其方法必須被子類別實現。

implements 實現

指定要實作之介面，使其行為具體化。

類別型別

Interface 介面

約束類別的行為，只描述屬性、方法。

屬性

  • 可選屬性 name?: string

  • 唯獨屬性 readonly name?: string：賦值後即不可變，類似於 const。

  • 任意屬性 [name: string: any]

    💡一旦定義任意屬性，其他所有屬性都必須為其子屬性。

介面 v.s. 型別別名

介面 型別別名
創建新名稱 ✅ ❌
代表型別 object 任何
同名多次定義 ✅ ❌
擴展 extends、union、intersection union、intersection

[React] 組件 Component

2022-06-25

定義

可將 UI 拆分成獨立且重複使用的程式碼

建立

  1. Function

    React 16.8 新增 Hook 功能，關注的是重新 render 後要做什麼。

    • 可用 Hooks
    • 沒有生命週期
    • 沒有 this，但可透過 useState() 使用 state。
    • props 傳遞透過 function 參數傳入

  2. Class

    用 ES6 class 來定義 component，關注的是生命週期。

    • 無法使用 Hooks
    • 有 this 指向，可用 this.state，擁有自己的 private data。
    • props 傳遞透過 this.porps

  3. creactClass（React 16 以後不支援）

[React] 生態圈

2022-06-25

React

JavaScript UI 庫

優點

  • 宣告式 Declarative：以 Virtual DOM 概念進行綁定、渲染
  • 高效率重用[組件](React－組件 Component.md)
  • 與 Backend for Frontend 相性良好
  • 可開發原生 APP ➪ React Native
  • 強大的社群生態系

Virtual DOM：以 JS 物件模擬 DOM 結構而產生的樹狀結構，不直接操作 DOM，而是暫存再更新 ➪ 提升效能

ReactDOM：把 Virtual DOM 渲染至文檔中變成實際瀏覽器的 DOM（= React.js App 進入點）

渲染機制

JSX（選用）

JS 語法擴展，也是一種語法糖 Syntatic Sugar。

語法糖：此語法對語言的功能無影響，但便於程式設計師使用。

限制

  • 只能有一個根元素 root
  • 駝峰式命名
  • 所有元素都須閉合
  • 保留字
    • class → className
    • for → htmlFor
  • 以 { } 篩寫 JS 程式碼
    • 行內樣式
    • 表達式
    • 屬性 attr
    • 綁定事件
    • 疊代元素

優點

  • 更加語意化且易懂的標籤
  • 實現宣告式 Declarative（注重 what to），而非命令式 Imperative（注重 how to）之寫法 ➪ 關注點分離
  • 結合原生 JS 語法

Node Package Manager（NPM）

Node.js 下的主流套件管理工具

Babel

將 ES6+ 、JSX 等程式碼轉換成不支援的瀏覽器能看懂的語法

useEffect 副作用控制

React 每次更新完 DOM 後觸發 useEffect

useEffect(callback, arr)

Param

  • callback function：定義 componentDidMount／componentDidUpdate 要做的事
  • array：當哪些 state、props 改變時，重新觸發 useEffect。

Rules

  • 宣告於最外層 {}
  • 僅用於 function component、自訂 hook

Use in

  • 手動操作 DOM
  • add／removeEventListenser
  • set／clearInterval
  • 使用外部函式庫
  • 向伺服器請求資料
